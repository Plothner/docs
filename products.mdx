Quanterios covers the full cryptographic governance lifecycle through five products that work together as a platform. You can adopt each product independently or deploy them together for end-to-end coverage: stop new debt at the pull request, discover what already exists, score and prioritize the risk, migrate safely, and then maintain agility as algorithms evolve.

## Available Now

<CardGroup cols={1}>
  <Card
    title="Crypto Debt Firewall"
    icon="shield-check"
    href="/customer/quickstart"
  >
    The Crypto Debt Firewall integrates with GitHub, GitLab, and Bitbucket CI pipelines to scan every pull request for cryptographic anti-patterns before they reach production. It combines Semgrep static analysis with an LLM-powered AI auditor that interprets findings in context, assigns a confidence score, and posts a structured review comment directly on the pull request. Developers receive actionable guidance without leaving their workflow. A free tier is available with no infrastructure required.
  </Card>
  <Card
    title="Crypto Risk Intelligence"
    icon="chart-bar"
    href="/customer/products#crypto-risk-intelligence"
  >
    Crypto Risk Intelligence discovers cryptographic assets across 40 connectors spanning AWS (12 services), Azure (8 services), GCP, Kubernetes, container registries, code repositories, and public certificate transparency logs. Each asset is scored using the HNDL model on a 0 to 100 scale across four dimensions: algorithm weakness, data sensitivity, network exposure, and migration urgency. Findings are mapped to 13 compliance frameworks including NIST CSF, CNSA 2.0, ISO 27001, SOC 2, HIPAA, and FedRAMP. Board-ready PDF reports are generated on demand.
  </Card>
  <Card
    title="Migration Intelligence"
    icon="arrow-right-arrow-left"
    href="/customer/products#migration-intelligence"
  >
    Migration Intelligence covers 29 supported migration paths and generates Terraform, Ansible, and Helm playbooks tailored to your environment. A Python code patcher applies hybrid shims that let old and new algorithms run side by side during the transition period. Breakage prediction uses an XGBoost model trained on real-world migration data, delivering an AUC of 0.94. Remediated findings are delivered as pull requests directly to your repository, keeping engineers in the loop without requiring them to manage the migration manually.
  </Card>
</CardGroup>

## How the Products Work Together

<Steps>
  <Step title="Prevent: Stop New Debt at the Source">
    The Crypto Debt Firewall sits at the CI layer and blocks cryptographic anti-patterns from merging. RSA key generation, weak PRNG usage, deprecated cipher suites, and hard-coded secrets are caught at the pull request stage before they enter your codebase or infrastructure definitions.
  </Step>
  <Step title="Discover: Build Your Cryptographic Inventory">
    Crypto Risk Intelligence connects to your cloud accounts, container registries, certificate stores, and code repositories through 40 purpose-built connectors. It builds a continuously updated inventory of every cryptographic asset across your environment, including assets in systems you did not know were in scope.
  </Step>
  <Step title="Fix: Migrate with Confidence">
    Migration Intelligence takes the prioritized findings from Crypto Risk Intelligence and generates the specific playbooks needed to remediate each one. Breakage prediction runs before any changes are applied. Hybrid mode keeps services running during the transition. Pull requests deliver the changes directly to your repositories for review and merge.
  </Step>
  <Step title="Integrate: Maintain Algorithm Agility">
    Once the Crypto Agility API is available, new services integrate through a single endpoint. Algorithm selection, rotation, and deprecation are managed at the API layer, decoupling your application code from cryptographic implementation details. As standards evolve, you update policy rather than code.
  </Step>
  <Step title="Issue: Operate a Post-Quantum PKI">
    The PQC Certificate Authority closes the loop by giving your organization the ability to issue ML-KEM and ML-DSA certificates internally. Combined with the Crypto Agility API, this enables a fully managed post-quantum cryptographic infrastructure with centralized lifecycle governance.
  </Step>
</Steps>
