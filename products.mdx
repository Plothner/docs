Quanterios covers the full cryptographic governance lifecycle through three stages that work together as a platform. You can adopt each stage independently or deploy them together for end-to-end coverage. Stop new debt at the pull request, discover and score what already exists, then migrate safely to quantum-safe algorithms.

## Stage 1: Assess

Understand everything that exists in your cryptographic environment before you can fix it.

<Card
  title="Cryptographic Risk Assessment"
  icon="chart-bar"
  href="/customer/risk-intelligence/overview"
>
  Connect 38 agentless connectors across AWS (12 services), Azure (8 services), GCP, Kubernetes, container registries, code repositories, and public certificate transparency logs. Every discovered asset is catalogued in a Cryptographic Bill of Materials and scored using the HNDL model on a 0 to 100 scale across four dimensions: data sensitivity, quantum timeline, exposure surface, and regulatory pressure. Findings are mapped to 13 compliance frameworks including CNSA 2.0, NIST FIPS 203/204/205, PCI DSS 4.0, HIPAA, and NIS2. Board-ready PDF reports are generated on demand. Powered by Quanterios Crypto Risk Intelligence.
</Card>

**What you can do in this stage:**

- Discover every TLS certificate, cloud encryption key, SSH key, and cryptographic code call across your infrastructure
- Generate a complete CBOM with algorithm, key size, expiry, and ownership metadata for every asset
- Score each asset against the Harvest Now, Decrypt Later threat model
- Map compliance gaps against 13 regulatory frameworks with deadline tracking
- Generate executive briefings and board-ready PDF reports

## Stage 2: Migrate

Convert findings into runnable fixes and deploy them safely.

<Card
  title="Migration Planning and Execution"
  icon="arrow-right-arrow-left"
  href="/customer/migration/overview"
>
  Select from 29 pre-built migration paths covering classical hygiene fixes, quantum-safe algorithm upgrades, and hybrid dual-algorithm deployments. The platform generates Terraform, Ansible, and Helm playbooks tailored to your environment. A Python code patcher applies hybrid shims that let old and new algorithms run side by side during the transition period. Breakage prediction uses an XGBoost model trained on 5,000+ real-world migration events, delivering an AUC of 0.94. Remediated findings are delivered as pull requests directly to your repository, keeping engineers in the loop without requiring them to manage the migration manually. Powered by Quanterios Migration Intelligence.
</Card>

**What you can do in this stage:**

- Choose from 13 classical, 10 quantum, and 6 hybrid migration paths
- Generate fully parameterized Terraform, Ansible, or Helm playbooks
- Predict breakage probability before any change reaches production
- Apply hybrid shims to application code so old and new algorithms coexist during rollover
- Monitor canary deployments with log-based cipher suite and error rate tracking
- Receive the completed migration as a pull request for review and merge

## Stage 3: Prevent

Stop new cryptographic debt before it enters the codebase.

<Card
  title="Continuous Crypto Debt Prevention"
  icon="shield-check"
  href="/customer/firewall/overview"
>
  The prevention stage runs at the CI layer on every pull request. A curated set of Semgrep rules catches known-weak primitives, deprecated functions, and insecure configurations with exact file and line references. An AI auditor analyzes the same diff for higher-order patterns that static rules cannot reliably detect: custom crypto implementations, IV reuse across call sites, and algorithmic downgrades. Findings are posted as a structured review comment directly on the pull request. Integrates with GitHub, GitLab, and Bitbucket. A free tier is available with no credit card required. Powered by the Quanterios Crypto Debt Firewall.
</Card>

**What you can do in this stage:**

- Scan every pull request for cryptographic vulnerabilities automatically at CI time
- Catch weak primitives, deprecated functions, and insecure cipher modes with deterministic rules
- Detect custom crypto implementations, IV reuse, weak RNG usage, and algorithmic downgrades with AI analysis
- Receive actionable remediation guidance without leaving the pull request workflow
- Generate training signals that feed back into your overall cryptographic risk profile

## How the Stages Work Together

<Steps>
  <Step title="Prevent: Stop New Debt at the Source">
    The continuous prevention stage sits at the CI layer and blocks cryptographic anti-patterns from merging. RSA key generation, weak PRNG usage, deprecated cipher suites, and hard-coded secrets are caught at the pull request stage before they enter your codebase or infrastructure definitions.
  </Step>
  <Step title="Assess: Build Your Cryptographic Inventory">
    The risk assessment stage connects to your cloud accounts, container registries, certificate stores, and code repositories through 38 purpose-built connectors. It builds a continuously updated inventory of every cryptographic asset across your environment, including assets in systems you did not know were in scope.
  </Step>
  <Step title="Migrate: Fix With Confidence">
    The migration stage takes the prioritized findings from your assessment and generates the specific playbooks needed to remediate each one. Breakage prediction runs before any changes are applied. Hybrid mode keeps services running during the transition. Pull requests deliver the changes directly to your repositories for review and merge.
  </Step>
  <Step title="Integrate: Maintain Algorithm Agility (Coming Soon)">
    Once the Crypto Agility API is available, new services integrate through a single endpoint. Algorithm selection, rotation, and deprecation are managed at the API layer, decoupling your application code from cryptographic implementation details. As standards evolve, you update policy rather than code.
  </Step>
  <Step title="Issue: Operate a Post-Quantum PKI (Coming Soon)">
    The PQC Certificate Authority closes the loop by giving your organization the ability to issue ML-KEM and ML-DSA certificates internally. Combined with the Crypto Agility API, this enables a fully managed post-quantum cryptographic infrastructure with centralized lifecycle governance.
  </Step>
</Steps>
