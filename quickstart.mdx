# Quickstart

Quanterios covers three stages: **assess** your cryptographic risk, **migrate** to stronger algorithms, and **prevent** new crypto debt. This guide walks through each stage.

<Info>
  You can start with any stage. Most enterprise customers begin with assessment. Developers often start with prevention (the free Firewall).
</Info>

## 1. Assess Your Crypto Risk

Connect your cloud infrastructure and discover every cryptographic asset.

<Steps>
  <Step title="Connect Your Cloud Account">
    Use the CBOM API to register your cloud assets. For AWS, provide an IAM role ARN with read-only access:

    ```bash
    curl -X POST https://api.quanterios.com/cbom/assets \
      -H "Content-Type: application/json" \
      -H "X-Tenant-Id: YOUR_TENANT_ID" \
      -H "X-API-Key: YOUR_API_KEY" \
      -d '{
        "asset_type": "tls_cert",
        "algorithm": "RSA-2048",
        "key_size": 2048,
        "location": "arn:aws:acm:us-east-1:123456789:certificate/abc",
        "exposure": "public",
        "sensitivity": "high",
        "discovered_by": "manual"
      }'
    ```

<Note>
  Replace `YOUR_TENANT_ID` and `YOUR_API_KEY` with your actual credentials. You can find these in your Quanterios account settings.
</Note>
  </Step>
  <Step title="View Your HNDL Risk Score">
    Every asset gets an AI-powered risk score from 0 to 100, measuring the Harvest Now, Decrypt Later threat:

    ```bash
    curl https://api.quanterios.com/api/v1/assets/ASSET_ID/hndl \
      -H "X-Tenant-Id: YOUR_TENANT_ID" \
      -H "X-API-Key: YOUR_API_KEY"
    ```

    The response includes four dimension scores (data sensitivity, quantum timeline, exposure surface, regulatory) and an AI-generated briefing explaining the risk.
  </Step>
  <Step title="Check Compliance">
    See how your assets map against 13 regulatory frameworks:

    ```bash
    curl https://api.quanterios.com/api/v1/assets/ASSET_ID/compliance \
      -H "X-Tenant-Id: YOUR_TENANT_ID" \
      -H "X-API-Key: YOUR_API_KEY"
    ```
  </Step>
</Steps>

## 2. Migrate to Stronger Algorithms

Preview a migration plan with breakage prediction:

<Steps>
  <Step title="Preview the Migration">
    ```bash
    curl -X POST https://api.quanterios.com/api/v1/migrate/preview \
      -H "Content-Type: application/json" \
      -H "X-Tenant-Id: YOUR_TENANT_ID" \
      -H "X-API-Key: YOUR_API_KEY" \
      -d '{
        "asset": {
          "algorithm": "RSA-2048",
          "asset_type": "tls_cert",
          "location": "arn:aws:acm:us-east-1:123456789:certificate/abc"
        },
        "cloud_provider": "aws"
      }'
    ```

    The response includes the target algorithm, breakage probability, confidence level, and a business impact assessment.
  </Step>
  <Step title="Generate the Playbook">
    Create a migration plan that produces runnable Terraform, Ansible, or Helm code:

    ```bash
    curl -X POST https://api.quanterios.com/api/v1/assets/ASSET_ID/migrate/plan \
      -H "Content-Type: application/json" \
      -H "X-Tenant-Id: YOUR_TENANT_ID" \
      -H "X-API-Key: YOUR_API_KEY" \
      -d '{"delivery": "zip"}'
    ```

    Download the ZIP bundle containing all migration steps, rollback procedures, and monitoring configuration.
  </Step>
</Steps>

## 3. Prevent New Crypto Debt

Install the Crypto Debt Firewall to catch cryptographic anti-patterns in every pull request:

<Steps>
  <Step title="Add the GitHub Action">
    Create `.github/workflows/quanterios-firewall.yml`:

    ```yaml
    name: Quanterios Crypto Debt Firewall
    on:
      pull_request:
        types: [opened, synchronize]
    permissions:
      contents: read
      pull-requests: write
    jobs:
      scan:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
            with:
              fetch-depth: 0
          - uses: Plothner/firewall-action@v1
            with:
              backend_url: https://api.quanterios.com
              github_token: ${{ secrets.GITHUB_TOKEN }}
    ```
  </Step>
  <Step title="Open a Pull Request">
    The Firewall scans automatically and posts findings as a PR comment, combining Semgrep rules with AI-powered analysis.
  </Step>
</Steps>

  For GitLab or Bitbucket setup, see the [Prevention](/firewall/overview) section.
</Tip>
