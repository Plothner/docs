<Info>
  Discovery is **entirely agentless**. Quanterios calls cloud provider APIs and on-premises endpoints using read-only credentials. Nothing is installed in your environment, and no production process is modified or restarted.
</Info>

## How discovery works

Quanterios uses the native APIs of each platform to enumerate cryptographic assets. For cloud providers this means API calls with read-only IAM or service account credentials. For on-premises systems it means secure API connections to PKI managers, LDAP directories, and source code repositories.

Discovery runs on first connection, then on a configurable schedule (default: every 24 hours). You can also trigger a manual scan at any time from the dashboard or via the API.

## Asset deduplication

The same certificate or key may appear in multiple connectors. For example, an ACM certificate may also be attached to a CloudFront distribution and a load balancer. Quanterios deduplicates assets by fingerprint (certificate SHA-256, KMS key ARN, etc.) so each physical asset appears exactly once in your CBOM, with all associated resources linked as references.

## Discovery permissions model

All connectors use the **principle of least privilege**. Each platform section documents the exact permissions required. Quanterios never requests write permissions, and credentials are stored encrypted at rest using AES-256-GCM.
